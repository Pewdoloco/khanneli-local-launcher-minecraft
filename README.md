# VanillaLauncher — движок

Собственный Windows-лаунчер для приватного Minecraft-сервера. Два режима одного
приложения: **Client** (докачка модов) и **Admin** (управление сервером). Один движок —
много модпак-деплойментов: сам по себе этот репозиторий не привязан ни к какому конкретному
модпаку, репозиторию модов или серверу.

Как это устроено и почему — см. `docs/ARCHITECTURE.md`, `docs/SPEC.md` (исходная
постановка), `docs/TASK_PATH_AUTODETECT.md` (модель "движок/модпак", автоопределение
путей, экран "Настройки").

## Что тут, а что нет

- **Здесь:** только исходный код (`src/`, `tests/`, `tools/`) и по-настоящему пустой
  дефолтный конфиг (`appsettings.json`/`appsettings.default.json` без единого значения
  конкретного модпака).
- **Не здесь:** моды, `manifest.json`, конфиги сервера — это хостит отдельный
  репозиторий конкретного модпака через GitHub Releases (см. пример ниже).

## Как адаптировать под свой модпак

1. Собери exe (`dotnet publish` — см. `docs/ARCHITECTURE.md`, раздел "Подготовка к
   релизу") или возьми готовый из [Releases](../../releases) этого репозитория
   (тег `engine-vX.Y.Z`).
2. Запусти, зайди в Admin-режим (первый запуск сам предложит задать пароль) → кнопка
   **"Настройки"**.
3. Укажи `ManifestUrl`/`GitHubOwner`/`GitHubRepo` своего репозитория-модпака, пути
   клиента/сервера (или их имена для автопоиска), `ServerExcludeMods` и остальные поля.
   Отдельно `EngineGitHubOwner`/`EngineGitHubRepo` — обычно `Pewdoloco`/
   `khanneli-local-launcher-minecraft` (этот репозиторий) — включает кнопку "Проверить
   обновление лаунчера" в клиентском окне: exe умеет само себя обновлять до новой версии
   движка, без участия админа.
4. Дальше публикация обновлений, старт/стоп сервера, бэкапы мира — обычный Admin-режим,
   уже настроенный под твой модпак.

## Пример реального деплоя на этом движке

[`Pewdoloco/VanillaLauncher-localServer`](https://github.com/Pewdoloco/VanillaLauncher-localServer) —
модпак VanillaScary: только `manifest.json`/моды через Releases и доки под конкретный
сервер (`docs/ADMIN_GUIDE.md`, `docs/CLIENT_GUIDE.md`), без единой строчки кода.

## Релизы движка

Тег `engine-vX.Y.Z` (`.github/workflows/engine-release.yml`) собирает self-contained
`VanillaLauncher.exe` и публикует его в Releases этого репозитория — отдельная нумерация,
не пересекается с версиями релизов конкретных модпаков.

## История

`CHANGELOG.md` и `docs/TASKS.md` — история разработки самого движка, включая часть до
разделения репозиториев (когда код и контент модпака VanillaScary были смешаны в одном
репозитории `VanillaLauncher-localServer`) — упоминания VanillaScary/Pewdoloco там
исторические, не конфигурация.
