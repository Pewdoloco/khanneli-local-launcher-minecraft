# HANDOFF — состояние на 2026-07-23, как продолжить в новой сессии

Этот файл — снимок состояния проекта для новой сессии Claude Code (или для себя через
пару месяцев). Если начинаешь новую сессию — открой этот файл, `docs/ARCHITECTURE.md`
и `docs/TASKS.md`, дальше не нужно пересказывать весь путь заново.

## Что это вообще такое (30 секунд)

Два репозитория:
- **[khanneli-local-launcher-minecraft](https://github.com/Pewdoloco/khanneli-local-launcher-minecraft)**
  (этот репозиторий) — "движок": весь код лаунчера, ни к какому конкретному модпаку не
  привязан. Публикует свои релизы `engine-vX.Y.Z` — самодостаточный exe.
- **[VanillaLauncher-localServer](https://github.com/Pewdoloco/VanillaLauncher-localServer)**
  — "модпак"-репозиторий для VanillaScary. Кода там нет вообще, только контент.

Один exe, два режима: **Client** (игрок — докачка модов, самообновление лаунчера) и
**Admin** (владелец сервера — публикация, старт/стоп сервера, конфигурация).

## Текущая версия

`EngineVersion.Current` = **1.4.0** (`src/VanillaLauncher.Client/EngineVersion.cs`). Теги
`engine-v1.3.0`, `engine-v1.3.1`, `engine-v1.4.0` все запущены через CI и опубликованы
успешно (проверено через GitHub Actions API до конца — `conclusion: success` на всех трёх).

## Хронология трёх релизов этой волны сессий

**`engine-v1.3.0`** — первая настоящая живая публикация на реальном модпаке (VanillaScary,
через тестовый репозиторий `Pewdoloco/test-pack`) нашла и починила три реальных бага:
1. `GitHubRepo`/`EngineGitHubRepo` принимали полную ссылку вместо голого имени репозитория
   (исходная жалоба пользователя, 404 на публикации) — фикс `GitHubRepoNameNormalizer`.
2. GitHub переименовывает ассеты релиза при сохранении (пробелы → точки) — манифест строил
   ссылку заранее из исходного имени, не совпадало с реальным — фикс: `UploadAssetAsync`
   возвращает настоящий `browser_download_url`.
3. Мод-пикер терял `_` из имён файлов на экране (WPF-мнемоники) — фикс экранированием.

Плюс фичи: автоопределение client-only модов (`FabricModEnvironmentReader`, ловит честные
`"environment": "client"`, не ловит лгущие `"*"` типа `better_tab`), разделение `GuideWindow`
по ролям, `GitHubApiErrorTranslator` (человекочитаемые подсказки поверх ошибок GitHub API).

**`engine-v1.3.1`** — `AppConfig.Load()` при отсутствующем appsettings.json работал только
в памяти, никогда не писал встроенный дефолт на диск — "удали appsettings.json и
перезапусти" из ClientGuide не работало (файл не появлялся). Фикс: `Load()` сам дописывает
дефолт на диск при отсутствии файла (best-effort, не роняет запуск без прав на запись).

**`engine-v1.4.0`** — самостоятельная настройка клиента без Admin-режима, по запросу
пользователя ("админу не пересылать файлы по тысяче раз"):
- `ClientSetupWindow` — лёгкое окно из `MainWindow` (кнопка "Настроить лаунчер", без
  пароля) — просит только GitHubOwner/GitHubRepo (+опционально ClientFolderName), сам
  считает ManifestUrl по стандартной формуле. Не секретные данные — не требует Admin-гейта.
- `EngineRepositoryDefaults` — канонический `Pewdoloco`/`khanneli-local-launcher-minecraft`
  как фоллбек для EngineGitHubOwner/EngineGitHubRepo, если не заданы — самообновление
  работает из коробки для любого модпака на этом движке.
- Более подробные описания полей автопоиска (имя папки vs путь, не рекурсивно) в Настройках.

Всего 109 тестов (71 Client + 38 Admin), все проходят на каждом релизе.

## Полностью реализовано и проверено живым прогоном

Всё из предыдущих HANDOFF плюс:
- Публикация модпака целиком (пустой репозиторий, токен без прав, переименование ассетов
  GitHub, клиентские моды в серверной сборке) — пройдена живьём несколько раз подряд.
- Автоопределение client-only модов, разделение инструкции по ролям, читаемые ошибки GitHub,
  самонастройка клиента без пароля, самовосстановление appsettings.json.

## Открытые вопросы / что осталось (в порядке убывания приоритета)

1. **Аудит `ServerExcludeMods` частично автоматизирован** — не ловит моды, лгущие в
   метаданных (`"*"`, но на деле клиентские — как `better_tab`). Принципиально не
   автоматизируется полностью, гибридный процесс и останется.
2. **Проверка целостности скачанного exe при самообновлении** (SHA-256/подпись) — не
   реализована.
3. **Цифровая подпись exe** — не сделана; для приватного сервера на несколько друзей,
   вероятно, не стоит стоимости платного сертификата. Существующий воркэраунд
   ("Подробнее → Выполнить в любом случае") задокументирован в guide-текстах.
4. **Лицензия репозитория** — не блокер для приватного модпака среди друзей, но модпак
   ре-публикует чужие мод-jar'ы — лицензии конкретных модов (не GitHub-лицензия репозитория)
   могут запрещать републикацию вне официальной площадки. Проверять индивидуально по модам.
5. **Привязка Admin-пароля к конкретному ПК** — не сделана, осознанно.

## Важно знать перед продолжением

- **`test-pack` (github.com/Pewdoloco/test-pack) — удалён или стал приватным** между
  сессиями (проверено: `GET /repos/Pewdoloco/test-pack` → 404 по состоянию на 2026-07-23).
  Если нужен тестовый прогон публикации — создать новый тестовый репозиторий заново (см.
  подход в истории чата: пустой репозиторий → залить первый коммит через `git push` →
  настроить GitHubOwner/GitHubRepo → публикация).
- **Файловая структура на машине администратора менялась между сессиями без ведома
  ассистента** — например, `D:\Modding\Servers\Tests` в какой-то момент превратился в
  `Tests\Client`/`Tests\Launcher`/`Tests\Server` с exe в `Tests\Launcher`. Не полагайся на
  пути из истории чата не глядя — проверь `ls`/`Get-ChildItem` заново в начале сессии.
- Для живого прогона (сборка + запуск + проверка через UI Automation) использовался
  PowerShell + `System.Windows.Automation` — хелперы не сохранены в репозитории (временный
  скрипт для тестирования, не часть продукта), восстанавливать по описанию в истории чата
  при необходимости. Для нативных MessageBox-кнопок без `InvokePattern` (некоторые системные
  диалоги "Да"/"Нет" экспонируются как `Pane`, не `Button`) нужен fallback через реальный
  клик мышью по координатам (`SetCursorPos`+`mouse_event`), не через UI Automation Invoke.

> Прочитай docs/HANDOFF.md, docs/ARCHITECTURE.md и docs/TASKS.md в репозитории

Локальный конфиг для тестов (`appsettings.json` в build output на машине администратора) —
тестовый и не коммитится, не путать с `src/VanillaLauncher.Client/appsettings.json`
(loose dev-копия в репозитории движка, пустая) или с реальным конфигом на боевой машине.
