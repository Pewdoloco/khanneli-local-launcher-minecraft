## Как проверялась работа в прошлых сессиях (нет своего CLI-тула, только это)

Для живого прогона (сборка + запуск + клики) использовался PowerShell +
`System.Windows.Automation`. Рабочий скрипт этой сессии — временный, не в репозитории,
пересобирался заново по описанию:
`Add-Type -AssemblyName UIAutomationClient, UIAutomationTypes` + `Find-ByName`/
`Find-ByAutomationId`/`Invoke-Element` (`InvokePattern`)/`Select-RadioElement`
(`SelectionItemPattern`)/`Get-TextBoxValue` (`ValuePattern`). Важные нюансы, найденные в
этот раз:
- **Окна, открытые с `Owner = this`, оказываются в дереве UI Automation детьми
  окна-владельца, а не корня рабочего стола** — `AutomationElement.RootElement.FindFirst`
  их не находит; искать нужно и как ребёнка окна-владельца, и (fallback) как ребёнка
  root — см. `GuideWindow`, открытое из `MainWindow`/`AdminWindow` (`Owner` всегда
  проставлен). `AdminWindow` открывается без `Owner` (`new AdminWindow(_config).Show()`),
  поэтому оно — прямой ребёнок root.
- **`PasswordBox` блокирует `ValuePattern`** — пароль вводится через `SetFocus()` +
  `System.Windows.Forms.SendKeys::SendWait(...)`, не через `ValuePattern.SetValue`.
- **Скрипты обязательно сохранять как UTF-8 с BOM** (`New-Object System.Text.UTF8Encoding($true)`
  + `[System.IO.File]::WriteAllText`) — без BOM `powershell.exe` (Windows PowerShell 5.1,
  не `pwsh`) читает кириллицу как ANSI и падает с ошибками парсера на русских строковых
  литералах.
- **Копировать в рабочую папку весь `bin/Debug/net8.0-windows`, не только `.exe`** — framework-
  dependent сборка без остальных `.dll`/`.deps.json`/`.runtimeconfig.json` падает сразу при
  старте с "The application to execute does not exist: ...VanillaLauncher.dll".
- GitHub Actions runs проверяются напрямую через
  `api.github.com/repos/Pewdoloco/khanneli-local-launcher-minecraft/actions/runs`
  (без токена, публичный репозиторий).

## Важно знать перед продолжением

- **`test-pack` (github.com/Pewdoloco/test-pack) — удалён/приватный.** Если нужен живой
  прогон публикации — создать новый тестовый репозиторий (публичный, с README при
  создании — иначе 422 "Repository is empty" на первой публикации).

  предупреждения** — не полагаться на пути из истории чата не глядя, перепроверять `ls`/
  `Get-ChildItem` в начале работы.
